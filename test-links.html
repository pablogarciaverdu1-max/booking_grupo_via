CRM Taller
Control de Citas y Clientes

Sistema de gestión que sincroniza con ambas páginas de booking y muestra todas las citas.

  • Lee reservas de gv_reservas, gv_reservas_madrid y gv_reservas_argentina
  • Muestra solicitudes pendientes de ambos talleres
  • Citas desde booking aparecen en MORADO (#a855f7)
  • Notificaciones en tiempo real
Abrir CRM
✓ Actualizado v1.28

📋 GUÍA DE PRUEBA

  1. Abre dos pestañas: Una con cada página de booking (Madrid y Argentina)
  2. En booking Madrid:
    • Selecciona fecha y hora
    • Rellena datos del cliente (nombre, teléfono, DNI)
    • Añade datos de la moto
    • Confirma la reserva
  3. En booking Argentina: Repite el mismo proceso (misma fecha/hora pero con cliente diferente)
  4. En CRM:
    • Verás un badge con "2 solicitudes pendientes"
    • Las citas aparecerán en MORADO en el calendario
    • Puedes aceptar/rechazar cada reserva
    • Se crearán como citas definitivas con flag "desde_booking"
  5. Verifica sincronización: Abre DevTools (F12) → Console → Escribe:
    localStorage.getItem('gv_reservas_madrid') // Ver reservas Madrid
    localStorage.getItem('gv_reservas_argentina') // Ver reservas Argentina
    localStorage.getItem('gv_ocupados_madrid') // Ver slots ocupados Madrid
⚠️ IMPORTANTE: Los datos se almacenan localmente en el navegador (localStorage). Si limpias el caché, perderás los datos. Para una base de datos persistente, es necesario usar Supabase, Firebase o similar.
